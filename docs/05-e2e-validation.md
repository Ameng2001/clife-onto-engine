# 05 · 端到端验证报告（真 Qwen 实证）

> 记录用**真实 LLM（qwen3.5-plus）** 对整栈做的端到端验证过程与结果。目的：把"能演示的机制"
> 压成"真 LLM 下能不能用、有没有价值"的证据。日期：2026-07-01。

---

## 1. 背景与目的

此前所有测试（143 项 pytest + 各 smoke）都是**离线、桩编译器**验证——证明机制对，但没答两个根本问题：

1. **真 LLM 上整栈 work 吗？** 中文口语的意图编译准不准、多跳查询、受治理写、回滚、审计能不能串通。
2. **知识供 LLM 推理有价值吗？** 附着知识不只是"进上下文"，是否**改变了 LLM 的选择**、让它领域正确。

为此搭了两个端到端 harness，**桩/真 Qwen 一键切**（有 `llm.local.json` 即走真）。

## 2. 方法与环境

- **环境**：`model=qwen3.5-plus`（DashScope/OpenAI 兼容端点，凭据在 gitignore 的 `llm.local.json`）。
- **数据**：真实感种子——两个盟市、不同乡土名录（巴彦淖尔：碱茅/星星草/披碱草；锡林郭勒：冰草/羊草），让治理判定有意义。
- **工具**：
  - `scripts/e2e_harness.py`：跑完整产品回路（口语 → 记忆接地+知识注入 → 意图编译 → 查 OQL/做 Action（guard→写后规则→确定性回滚→审计）→ 回写记忆）。
  - `scripts/e2e_knowledge_influence.py`：同一句知识敏感口语，`load_knowledge` 关/开各跑一次，`SpyCompiler` 录下喂给 LLM 的 `memory_text`，并排对比。

## 3. 测试一：端到端整栈回路（真 Qwen）

**剧本**（一段"施工方"会话，含治理兜底 money-shot）与**真 Qwen 实测结果**：

| # | 口语 | 结果 | 判读 |
|---|---|---|---|
| 1 | 巴彦淖尔有哪些地块？ | [查询] 1 行（parcel_001） | 意图编译→OQL 正确 |
| 2 | parcel_001 能用哪些修复方法？ | [查询] 1 行（喷播 RestorationMethod） | **真 LLM 走了完整多跳** Site→退化→修复方法（比桩强，桩只走一跳到退化） |
| 3 | 给 parcel_001 出一地一方，用碱茅，预算300 | [已执行] 写入 SeedPack/Project · 置信 1.0 | 合规（碱茅是巴彦淖尔乡土）→ 提交 |
| 4 | 给 parcel_001 出方案，用紫花苜蓿，预算300 | [本体兜底·拒绝] 违反「乡土合规」：非乡土 | 非乡土 → 确定性回滚 |
| 5 | 给 parcel_002 出方案，用碱茅，预算300 | [本体兜底·拒绝] 违反「乡土合规」：碱茅非锡林郭勒乡土 | **跨区域用错草种被兜住**（真实治理价值） |
| 6 | 帮我出个方案 | [澄清] 请提供 site_id/species/budget | 缺参 → 澄清（措辞比桩更自然完整） |

**汇总**：查询 2 · 提交 1 · 治理拒绝 2 · 澄清 1 · 错误 0 · 审计留痕 3 条（committed×1, rejected×2）。
**OAG 回路核心（既有合规提交、又有本体兜底拒绝）：✓ 成立。**

**结论**：整栈在真 Qwen 上端到端 work；中文口语意图编译 6/6 无误（多跳还比桩强）；**OAG 命门成立**——真概率模型提议、确定性本体边界兜住。

## 4. 测试二：知识影响力对比（真 Qwen）

**方法**：同一句判断类口语，`load_knowledge` 关（无知识）/ 开（有知识）各跑一次真 Qwen，并排看建议。`SpyCompiler` 确认知识确实到达 LLM 的 `memory_text`（管道 ✓）。

### 口语1：「parcel_001 是重度盐碱地，接下来该先做什么？」

- **[无知识]**：*"…建议立即调用『出一地一方』动作，输入 site_id/species/budget，以生成修复方案并派单。在方案生成前，切勿盲目施工。"*
  → **领域错误**：重度盐碱地直接上植被恢复方案是错的，该先改良土壤。LLM 缺知识时默认跳到"出方案"。
- **[有知识]**：*"根据退化分级处置手册，parcel_001 作为重度盐碱地，**不应直接进行植被恢复**。接下来的首要步骤是**实施工程改良或化学改良**措施，以降低土壤盐分、改善土壤结构。待土壤条件改良达标后，再进行**喷播乡土草种及后期封育**。"*
  → **完全正确的分阶段**，一字贴处置手册。

### 口语2：「这块地退化挺严重的，直接喷播碱茅行不行？」

- **[无知识]**：*"…直接喷播碱茅存在较大风险…建议先调用『出一地一方』…"* → 含糊、没正面答、仍往"出方案"引。
- **[有知识]**：*"**不行**。根据退化分级处置手册，该地块属于重度退化（盐碱化），不应直接进行植被恢复（如喷播碱茅）。首要步骤必须**先实施工程改良或化学改良**…待土壤条件改良达标后，才能进行喷播乡土草种及后期封育。"* → **清楚的"不行" + 正确理由**。

两句**行为差异均为"有"**（知识改变了 LLM）；管道 ✓（有知识那次 `memory_text` 含处置手册知识，无知识那次不含）。

**结论**：知识不只是改善、是**防错**——无知识时 LLM 给出**领域错误**建议（重度盐碱地直接上植被），有知识时**纠正**为处置手册的正确分阶段。知识层押注证实。

## 5. 一个被验证印证的分层洞察

结果同时验证了「多场景知识」的设计——**两种知识性质、两种机制，各归各位，都在真 Qwen 上工作**：

| 知识 | 性质 | 落点 | 表现 |
|---|---|---|---|
| 分阶段处置（先改良再喷播） | **建议型**（不强制） | 附着知识 → advise/Memory | 帮 LLM **想对**（口语1/2 有知识那次正确） |
| 乡土性（草种适配） | **强制型** | Rule | 写时**兜底**（测试一 #4/#5 非乡土/跨区域回滚） |

## 6. 总结论

三个核心论点在**真 Qwen** 上全部证实：

1. **整栈端到端 work**（测试一 6/6）。
2. **治理兜底成立**——LLM 提议非乡土/跨区域草种，本体确定性回滚（测试一 #4/#5）。
3. **知识改变并防止 LLM 领域错误**（测试二两句）。

> "它到底能不能用、有没有价值"——**答案是肯定的，且有真 LLM 实证背书。**

## 7. 诚实边界

- **定性、非统计**：测试一 6 步、测试二 2 句，是**存在性 + 防错级**证据，非大样本。结果干净、可复现、无歧义。
- **仍是种子数据**：真实感但非真业务数据（`tenants/mengcao` 尚为占位）。
- **未测**：对抗/刁钻说法的鲁棒性、延迟/成本。这些是"从能用到规模化"的下一程。
